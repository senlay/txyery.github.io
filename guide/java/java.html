# main方法是什么？

  • java中main方法为程序执行的入口，通过这个入口函数可以执行程序。
  • 通过观察，可以发现main方法是public的,静态的static修饰的，没有返回值，参数类型是String[]。
  • 然而，为什么要定义main这样的方法呢，为什么要public?因为这是Java的规范，jvm启动会首先执行main函数,你 如果改为private私有的，那么jvm是找不到main方法的。
  • 为什么不要返回值？如果有返回值，程序是不是就意味着结束了，那么这样的程序岂不是很短暂，还没开始就 return，而且jvm要你 返回值干嘛？
  • 为什么要static修饰？为了让jvm调用main方法更方便，不需要再实列化对象了，因为实列化对象时候会分为有参数和 无参数构造器，那么jvm怎么知道你是什么样的构造器呢？使用静态main方法避免了这个尬尴。
  • mian方法可以抛出异常，也可以捕获异常

# main方法之总结、归纳

通过学习，了解到Java程序启动时候，jvm会首先启动main方法，该方法即是整个程序的入口程序， 就好像大门一样，启动了这个方法就等于打开了Java程序的第一扇门，那么为什么要这样定义main方法呢？ 因为这是Java的规范，我给你一扇门你要问我为什么门是这么样子，那是不是有点强词夺理。 除此之外，main函数既是打开世界的一把钥匙，也是关闭它的钥匙。你可以hello world 也可以 stop. 当然，一扇门也有坏的时候，main方法也允许抛出异常，等人来修； main方法也可以自己吃掉异常，自己来修。

# Java开发环境配置

在"系统变量"中设置3项属性，JAVA_HOME,PATH,CLASSPATH(大小写无所谓),若已存在则点击"编辑"， 不存在则点击"新建"。 变量设置参数如下：

  • 变量名：JAVA_HOME
  • 变量值：C:\Program Files (x86)\Java\jdk1.8.0_91 // 要根据自己的实际路径配置
  • 变量名：CLASSPATH
  • 变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar; //记得前面有个"."
  • 变量名：Path
  • 变量值：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;

# 二、Java语言有什么特点？

Java语言诞生于1995年5月，由sun公司推出，目前，由Oracle公司维护。Java语言有以下特点：

  • 1.面向对象: Java语言是面向对象的语言，提供类和接口、继承等机制。可反映现实世界中的对象，如汽车-car、食物-food等
  • 2.分布式: Java语言支持Internet应用的开发，在基本的Java应用编程接口中有一个网络应用编程接口（java net），它提供了用于网络应用编程的类库，包括URL、URLConnection、Socket、ServerSocket等。Java的RMI（远程方法激活）机制也是开发分布式应用的重要手段。
  • 3.健壮: Java的强类型机制、异常处理、垃圾的自动收集等是Java程序健壮性的重要保证。对指针的丢弃是Java的明智选择。Java的安全检查机制使得Java更具健壮性。
  • 4.安全: Java通常被用在网络环境中，为此，Java提供了一个安全机制以防恶意代码的攻击。除了Java语言具有的许多安全特性以外，Java对通过网络下载的类具有一个安全防范机制（类ClassLoader），如分配不同的名字空间以防替代本地的同名类、字节代码检查，并提供安全管理机制（类SecurityManager）让Java应用设置安全哨兵。
  • 5.多线程: Java语言是多线程的，线程在Java中也是对象，封装了线程的基本属性和方法，如run方法，也提供了线程安全的机制，如synchronized关键字、lock锁机制
  • 6.可移植、跨平台: Java语言具有平台无关性，一次编译到处运行
  • 7.解释性语言: Java语言是解释型语言，需要先编译后运行
  • 8.动态的: Java程序需要的类能够动态地被载入到运行环境，也可以通过网络来载入所需要的类，如第三方jar、maven依赖等
  • 2019-11-04 记录：如果让自己结合自己的开发经验说下Java语言的特点该怎么说呢？：

对于Java语言的感触：大家一起工作，一起维护和开发软件，通过svn或者git协同作战，有利于团结一切力量创造价值。 Java语言也要求开发过程中要严格遵循Java规范，是其健壮性的体现。对于分布式，目前接触的还比较粗浅，需要再接再厉； 对于多线程，项目中多是启用多个线程去处理大批量数据，如数据归档，订单批量保存，异步操作，定时任务等， 当然，使用中会涉及到线程安全的问题，一般项目中是通过加锁保障程序的执行顺序。

# 三、Java语言数据类型

  • 整数型 ：

    • byte: 8位，有符号；取值范围 -128-127 ，默认为0
    • short：数据类型是16位；取值范围 -2^15-2^15 - 1，默认值是0
    • int：数据类型是32位；取值范围 -2^31-2^31 - 1，默认值是0
    • long：数据类型是64位；取值范围 -2^63-2^63 - 1，默认值是0L

  • 浮点型 ：

    • float：单精度 32位，默认值0.0f,如 float f1 = 1.2f。不能表示精确的值
    • double：双精度 64位,默认值是0.0d；如 double d1 = 123.4。不能表示精确的值

  • Boolean型 ：boolean：只有true和false，默认false

  • 字符型 ：char是一个单一的16位Unicode字符，char数据类型可以储存任何字符，如 char l = 'a'

  • 引用型数据类型：

    • 如对象、数据等，Integer、Double、Long、List
    • 所有引用类型的默认值都是null。
    • 引用类型变量由类的构造器创建，即new 对象创建。可以使用它们访问所引用的对象。这些变量在声明时就被指定为一个特定的类型，比如Employee、Book等。变量一旦声明后，类型就不能被改变了。

  • Java常量

    • 一般大写
    • 所谓常量，即是固定的值，一般用“final”修饰，修饰后不能再改变，是为终极审判，一旦使用final就固定了

  • 静态变量（类变量）

    • 即static修饰的变量，如 static String name = "sky";该变量一旦声名，即宣誓着属于类，可以通过类.属性名的方式调用
    • 类只拥有类变量的一份拷贝
    • 静态变量储存在静态存储区。经常被声明为常量，很少单独使用static声明变量。
    • 静态变量在程序开始时创建，在程序结束时销毁。
    • 与实例变量具有相似的可见性。但为了对类的使用者可见，大多数静态变量声明为public类型。
    • 默认值和实例变量相似。数值型变量默认值是0，布尔型默认值是false，引用类型默认值是null。变量的值可以在声明的时候指定，也可以在构造方法中指定。此外，静态变量还可以在静态语句块中初始化。
    • 静态变量可以通过：ClassName.VariableName的方式访问。
    • 类变量被声明为public static final类型时，类变量名称必须使用大写字母。如果静态变量不是public和final类型，其命名方式与实例变量以及局部变量的命名方式一致。

  • 局部变量

    • 局部变量声明在方法、构造方法或者语句块中；
    • 局部变量在方法、构造方法、或者语句块被执行的时候创建，当它们执行完成后，变量将会被销毁；
    • 访问修饰符不能用于局部变量；
    • 局部变量只在声明它的方法、构造方法或者语句块中可见；
    • 局部变量是在栈上分配的。
    • 局部变量没有默认值，所以局部变量被声明后，必须经过初始化，才可以使用。

  • 实列变量

    • 实例变量声明在一个类中，但在方法、构造方法和语句块之外；
    • 当一个对象被实例化之后，每个实例变量的值就跟着确定；
    • 生命周期：实例变量在对象创建的时候创建，在对象被销毁的时候销毁；
    • 实例变量的值应该至少被一个方法、构造方法或者语句块引用，使得外部能够通过这些方式获取实例变量信息；
    • 实例变量可以声明在使用前或者使用后；
    • 实例变量对于类中的方法、构造方法或者语句块是可见的。一般情况下应该把实例变量设为私有。通过使用访问修饰符可以使实例变量对子类可见；；
    • 实例变量具有默认值。数值型变量的默认值是0，布尔型变量的默认值是false，引用类型变量的默认值是null。变量的值可以在声明时指定，也可以在构造方法中指定；

# 数据类型小结：
